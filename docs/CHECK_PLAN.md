# PLAN: `FielderSlice` embeds `Fielder`

## Problem

`fmt.FielderSlice` and `fmt.Fielder` are unrelated interfaces. A slice type cannot
be passed to any function that accepts `fmt.Fielder` — including `json.Encode` and
`json.Decode` — without changing their signatures.

## Solution

`FielderSlice` embeds `Fielder`. Any type that implements `FielderSlice` automatically
satisfies `Fielder`. Existing call sites that accept `fmt.Fielder` receive slices
transparently; the callee type-asserts to `FielderSlice` to choose array encoding.

**No change to `Encode`/`Decode` signatures. No new public functions.**

---

## Change to `field.go`

```go
// BEFORE
type FielderSlice interface {
    Len() int
    At(i int) Fielder
    Append() Fielder
}

// AFTER
type FielderSlice interface {
    Fielder                  // Schema() []Field + Pointers() []any
    Len() int
    At(i int) Fielder
    Append() Fielder
}
```

---

## Impact on `FielderSlice` implementors

All types implementing `FielderSlice` must now also implement `Schema()` and
`Pointers()`. Generated `{Name}List` types (added by ormc) provide stubs that
return `nil` — they are never called for list encoding because the type switch
in `json.Encode` matches `FielderSlice` first:

```go
// generated by ormc
func (s *ReservationList) Schema() []fmt.Field { return nil }
func (s *ReservationList) Pointers() []any     { return nil }
```

Existing hand-written `FielderSlice` implementations (e.g., in test code) must
add the two stub methods. The compiler will flag any missing implementation.
