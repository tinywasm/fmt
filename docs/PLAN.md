# PLAN: Add `Field`, `FieldType`, and `Fielder` Interface

## Development Rules

- **Standard Library Only:** No external assertion libraries. Use `testing` and `reflect` for test helpers only.
- **Testing Runner:** Install and use `gotest`:
  ```bash
  go install github.com/tinywasm/devflow/cmd/gotest@latest
  ```
- **Max 500 lines per file.** If exceeded, subdivide by domain.
- **Flat hierarchy.** No subdirectories for library code.
- **Documentation First:** Update docs before coding.
- **Publishing:** Use `gopush 'message'` after tests pass and docs are updated.
- **TinyGo Compatible:** No `fmt`, `strings`, `strconv`, `errors` from stdlib.
- **No maps** in exported types (binary bloat in WASM).

## Context

The `tinywasm/fmt` package is the shared foundation for the entire tinywasm ecosystem. It already provides `Kind` (type system), `Convert` (type conversion), `KeyValue`, `Err` (errors), and string/parsing utilities.

This plan adds **schema metadata types** (`Field`, `FieldType`) and a **`Fielder` interface** so that downstream packages (`orm`, `form`, and others like `crudp`) can share a common contract for describing struct fields without runtime reflection.

### Why `fmt`?

- `fmt` is already a dependency of both `orm` and `form`.
- No circular dependencies are introduced.
- `Field`/`FieldType` are data description primitives — same abstraction level as `Kind` and `KeyValue`.

## What This Plan Does NOT Cover

- Changes to `orm` or `form` — those have their own independent plans.
- Foreign key fields (`Ref`, `RefColumn`) — those are database-specific and stay in `orm`.

## Stage 1: Add `FieldType` and `Field`

← None | Next → [Stage 2](#stage-2-add-fielder-interface)

### 1.1 Create `field.go`

New file: `field.go` (~50 lines)

```go
package fmt

// FieldType represents the abstract storage type of a struct field.
type FieldType int

const (
    FieldText FieldType = iota
    FieldInt
    FieldFloat
    FieldBool
    FieldBlob
)

// Field describes a single field in a struct's schema.
// It provides type metadata and constraint flags used by both
// database (orm) and UI (form) layers.
type Field struct {
    Name    string
    Type    FieldType
    PK      bool // Primary Key
    Unique  bool
    NotNull bool
    AutoInc bool // Auto-increment (numeric fields only)
}
```

**Naming rationale:**
- `FieldText` instead of `TypeText` to avoid collision with Go's `reflect` and to namespace under the `Field` domain.
- Bool flags instead of bitmask for readability — there are only 4 constraints.

### 1.2 Add `FieldType.String()` method

In the same `field.go`:

```go
var fieldTypeNames = []string{"text", "int", "float", "bool", "blob"}

func (ft FieldType) String() string {
    if int(ft) >= 0 && int(ft) < len(fieldTypeNames) {
        return fieldTypeNames[ft]
    }
    return "unknown"
}
```

### 1.3 Tests

New file: `field_test.go`

- `TestFieldTypeString`: Verify all 5 type names + out-of-range returns `"unknown"`.
- `TestFieldZeroValue`: Verify `Field{}` has `FieldText` type and all bools `false`.
- `TestFieldConstraints`: Create fields with various constraint combos, assert each bool independently.

### 1.4 Run tests

```bash
gotest
```

## Stage 2: Add `Fielder` Interface

← [Stage 1](#stage-1-add-fieldtype-and-field) | Next → [Stage 3](#stage-3-documentation-and-publish)

### 2.1 Add interface to `field.go`

Append to the existing `field.go`:

```go
// Fielder describes any type that can expose its schema, values, and
// writable pointers. It is the shared contract between orm (database)
// and form (UI) layers, replacing runtime reflection.
//
// Implementations are generated by ormc — never written by hand.
//
// Contract:
//   - Schema(), Values(), and Pointers() MUST return slices of the same length.
//   - The i-th element in each slice corresponds to the same struct field.
//   - Values() returns current field values for reading.
//   - Pointers() returns pointers to fields for writing (scan/sync).
type Fielder interface {
    Schema() []Field
    Values() []any
    Pointers() []any
}
```

### 2.2 Tests

Add to `field_test.go`:

- `TestFielderInterface`: Create a mock struct implementing `Fielder` manually. Verify:
  - `len(Schema()) == len(Values()) == len(Pointers())`
  - Schema field names match expected values.
  - Writing through `Pointers()` changes what `Values()` returns.

Example mock:

```go
type mockUser struct {
    id   string
    name string
}

func (m *mockUser) Schema() []fmt.Field {
    return []fmt.Field{
        {Name: "id", Type: fmt.FieldText, PK: true},
        {Name: "name", Type: fmt.FieldText, NotNull: true},
    }
}
func (m *mockUser) Values() []any  { return []any{m.id, m.name} }
func (m *mockUser) Pointers() []any { return []any{&m.id, &m.name} }
```

### 2.3 Run tests

```bash
gotest
```
