# Field and Fielder API

The `field` API provides a standard way to describe struct schemas and access their values without using runtime reflection. This is essential for maintaining a small binary size in WebAssembly environments.

## FieldType

`FieldType` represents the abstract storage type of a struct field.

| Constant | String | Description |
|----------|--------|-------------|
| `FieldText` | `"text"` | String or text data |
| `FieldInt` | `"int"` | Integer numbers |
| `FieldFloat` | `"float"` | Floating point numbers |
| `FieldBool` | `"bool"` | Boolean values |
| `FieldBlob` | `"blob"` | Binary data |

### String()

Returns the string representation of the `FieldType`.

```go
fmt.FieldInt.String() // returns "int"
```

## Field

`Field` describes a single field in a struct's schema with its metadata and constraints.

```go
type Field struct {
    Name    string
    Type    FieldType
    PK      bool   // Primary Key
    Unique  bool
    NotNull bool
    AutoInc bool   // Auto-increment (numeric fields only)
    Input   string // UI hint: input type override ("email", "password", "-", etc.). Empty = auto.
}
```

### Input Hint Semantics

- `""` (empty) → form uses its name-based heuristic to pick input type (default behavior).
- `"email"`, `"password"`, `"textarea"`, `"date"`, etc. → form uses this as explicit input type override.
- `"-"` → form excludes this field entirely (not rendered in the form).

The `Input` hint is typically populated by `ormc` from the struct's `form:"..."` tag.

## Fielder Interface

The `Fielder` interface is the shared contract between various layers (like ORM and UI forms) to interact with structs.

```go
type Fielder interface {
    Schema() []Field
    Values() []any
    Pointers() []any
}
```

### Contract

- `Schema()`, `Values()`, and `Pointers()` MUST return slices of the same length.
- The i-th element in each slice corresponds to the same struct field.
- `Values()` returns current field values for reading.
- `Pointers()` returns pointers to fields for writing (scanning from database or syncing from forms).

### Example Implementation

Usually, these implementations are generated by `ormc`.

```go
type User struct {
    ID   string
    Name string
}

func (u *User) Schema() []fmt.Field {
    return []fmt.Field{
        {Name: "id", Type: fmt.FieldText, PK: true},
        {Name: "name", Type: fmt.FieldText, NotNull: true},
    }
}

func (u *User) Values() []any {
    return []any{u.ID, u.Name}
}

func (u *User) Pointers() []any {
    return []any{&u.ID, &u.Name}
}
```
